# v1.7.4 Public Config Hardening And Preview Hotkeys Design

**Date:** 2026-03-19

## Goal

Ship `1.7.4` as a patch release that removes working Tropicana credentials from public artifacts and adds a `Ctrl+F` preview shortcut to both the main label generation workflow and the rail labels workflow.

## Problem

The current public bundle path is no longer copying the repository root `.env`, but the shipped template source still contains working Tropicana DB credentials. As a result, public/package builds can still authenticate successfully against production, which breaks the intended security boundary and makes the dedicated Tropicana installer path moot.

Separately, the GUI workflows already support right-click terminal-style clipboard behavior, but operators still have to use the mouse to trigger preview actions in the two most common workflows. A keyboard shortcut would reduce friction during repetitive operation.

## Constraints

- This release must remain SemVer patch scope.
- Public builds must not contain functional DB credentials.
- Tropicana internal distribution must continue to support a one-file installer path and a fallback config-only script.
- The preview shortcut must work reliably while the main input field has focus.
- The design must be recorded as an ADR in addition to the feature spec.

## Root Cause Summary

Fresh packaged CLI smoke testing against the built `1.7.3` app image showed:

- `run.bat config` resolved config from the bundled install-local `wms-tags.env`
- `run.bat db-test` succeeded against `10.19.68.61:1521/WMSP`

That means the public runtime path is functional because `config/wms-tags.env.example` is still populated with real Tropicana DB values. The secret leak is therefore in the checked-in example config content, not only in bundle-copy behavior.

## Selected Approach

1. Replace `config/wms-tags.env.example` with truly non-functional dummy values.
2. Preserve the existing per-user Tropicana config precedence so internal installs still work after updates.
3. Fix the generated fallback Tropicana config installer path so embedded payloads remain usable.
4. Add a shared GUI shortcut binding for `Ctrl+F` and wire it to preview/load-preview in the two workflow windows.

## User Flow

### Public users

- Public portable/app-image/install artifacts still launch normally.
- Database-backed operations fail safely until the user supplies valid config.
- Public releases no longer ship working Tropicana DB access by default.

### Tropicana users

- Tropicana setup EXE remains the main install path.
- The fallback config installer script remains the repair/rotation path.
- Once Tropicana config is installed, updates continue to use the per-user config path.

### GUI operators

- In the main label generation workflow, pressing `Ctrl+F` triggers the same behavior as clicking `Preview`.
- In the rail labels workflow, pressing `Ctrl+F` triggers the same behavior as clicking `Load Preview`.
- The shortcut works while the train/shipment input field is focused.

## Technical Design

### Public config hardening

- Sanitize `config/wms-tags.env.example` so it cannot authenticate against production.
- Keep placeholders obvious and intentionally unusable until replaced.
- Add a smoke/regression test that proves a bundle built from public sources resolves dummy config and does not silently carry working credentials.

### Tropicana installer integrity

- Keep per-user config location under `%LOCALAPPDATA%\Tropicana\WMS-Pallet-Tag-System\wms-tags.env`.
- Ensure the generated fallback config installer script can use its embedded payload consistently.
- Keep Tropicana-only artifacts outside the public release workflow.

### Preview shortcut binding

- Extract or add a focused helper for workflow hotkey binding.
- Bind `Ctrl+F` at the window/root-pane level so the shortcut works while the main input field is focused.
- Reuse the existing preview/load-preview button action rather than duplicating workflow logic.
- If the target button is disabled, the shortcut should do nothing.

## Error Handling

- Public config remains invalid: `db-test` should fail with a configuration/auth error rather than silently succeeding.
- Missing Tropicana per-user config: packaged install behaves like a public install until configuration is applied.
- Disabled preview action: `Ctrl+F` should no-op instead of forcing inconsistent state.

## Testing Strategy

- Add/extend regression tests for config discovery and public bundle config seeding.
- Add a packaged CLI smoke test proving public app-image config is non-functional until Tropicana config is installed.
- Add GUI tests for the preview shortcut helper where feasible.
- Verify the fallback Tropicana config installer script still writes valid per-user config.

## Documentation

- Update release notes and README if needed to reflect that public example config is dummy-only.
- Add an ADR covering both the public-config boundary and the `Ctrl+F` workflow decision.

## SemVer

This remains a patch release because it fixes a release-packaging/security boundary bug and adds a narrow workflow shortcut without changing external APIs.
