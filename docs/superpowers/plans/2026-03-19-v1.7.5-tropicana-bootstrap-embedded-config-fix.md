# v1.7.5 Tropicana Bootstrap Embedded Config Fix Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the broken embedded-config handoff in the Tropicana bootstrap/support installer flow and prove the generated support script works end to end without an external `.env`.

**Architecture:** Start from `patch/v1.7.4-public-config-hotkeys`, bump to `1.7.5`, and keep the fix confined to the Tropicana installer script template plus regression coverage. The key guardrail is to test the generated `Install-Tropicana-Config.ps1` artifact itself, not just inspect its text for embedded base64.

**Tech Stack:** Maven multi-module Java 17 project, PowerShell packaging scripts, JUnit 5, Windows batch/PowerShell smoke verification.
