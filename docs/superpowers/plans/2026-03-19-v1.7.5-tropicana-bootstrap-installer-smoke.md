# v1.7.5 Tropicana Bootstrap Installer Smoke Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a release-only smoke path that performs a real local Tropicana bootstrap install, then verifies per-user config precedence and post-install runtime connectivity.

**Architecture:** Extend the existing smoke manifest and PowerShell harness with an opt-in installer scenario rather than changing the default fast smoke path. Reuse the current Tropicana builder and installer scripts, add only the minimum unattended/automation seam needed to install into isolated temp roots and verify the installed app through its real launcher.

**Tech Stack:** PowerShell, jpackage Windows installer, IExpress bootstrapper, existing Tropicana install scripts, Maven-built CLI/app image, Markdown docs.
