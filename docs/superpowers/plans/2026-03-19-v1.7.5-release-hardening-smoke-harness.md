# v1.7.5 Release Hardening Smoke Harness Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a deterministic smoke harness that verifies critical workflows in both repo/JAR and packaged-app modes, documents GUI coverage parity, and gates release tagging on smoke evidence.

**Architecture:** Add a shared smoke manifest plus a PowerShell harness that can execute the same scenarios against either a repo-built CLI jar or a packaged app image. Keep workflow logic in existing CLI/shared services, add only minimal smoke-only seams where required, and emit both human-readable and machine-readable reports for release validation.

**Tech Stack:** PowerShell, Maven, Java 17, existing CLI commands, existing GUI/shared workflow services, packaged app image scripts, Markdown docs.
