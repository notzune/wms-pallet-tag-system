# v1.7.2 Release And README Cleanup Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish release `v1.7.2` from the already-merged remote `main` commit line and sanitize the README so it no longer lists stale tracked issues.

**Architecture:** Use `origin/main` as the release source of truth, make the README cleanup as a small doc-only commit directly on top of that merged state, then tag the resulting commit as `v1.7.2`. Keep the local divergent `main` history untouched by doing the release work on a temporary branch rooted at `origin/main`.

**Tech Stack:** Git, GitHub remote, Markdown documentation
