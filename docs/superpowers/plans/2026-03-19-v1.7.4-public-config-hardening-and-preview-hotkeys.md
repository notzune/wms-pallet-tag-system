# v1.7.4 Public Config Hardening And Preview Hotkeys Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `1.7.4` from the existing Tropicana-installer branch line so public artifacts no longer contain functional Tropicana database settings and both workflow windows support a reliable `Ctrl+F` preview shortcut.

**Architecture:** Start from `patch/tropicana-installer-config`, not the current local `main`, because the Tropicana installer/config persistence work only exists on that branch. Keep public config safety centered on one dummy checked-in template and make both bundle scripts consume only that template. Extract a small Swing shortcut helper that binds `Ctrl+F` to an existing button action and no-ops when the action is disabled, then apply it to both workflow windows.

**Tech Stack:** Maven multi-module Java 17 project, Swing GUI, PowerShell build scripts, JUnit 5, Windows batch/PowerShell smoke verification.

```java
static void bindPreviewShortcut(JRootPane rootPane, AbstractButton button, String actionKey) {
    InputMap inputMap = rootPane.getInputMap(JComponent.WHEN_IN_FOCUSED_WINDOW);
    ActionMap actionMap = rootPane.getActionMap();
    inputMap.put(KeyStroke.getKeyStroke(KeyEvent.VK_F, InputEvent.CTRL_DOWN_MASK), actionKey);
    actionMap.put(actionKey, new AbstractAction() {
        @Override
        public void actionPerformed(ActionEvent e) {
            if (!button.isEnabled()) {
                return;
            }
            button.doClick(0);
        }
    });
}
```
