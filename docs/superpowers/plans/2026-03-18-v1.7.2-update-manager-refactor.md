# v1.7.2 Update Manager Refactor Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a release-catalog-driven update manager for stable and prerelease awareness, add guided upgrade/downgrade selection, improve installer UX, and refresh the placeholder app icon for `1.7.2`.

**Architecture:** Replace the current single-release check with a catalog, policy, and action split under `core.update`, then keep the Swing GUI as a thin consumer of update state and install actions. Extend the installer helper path to show visible progress and relaunch the app after successful install, and wire a new placeholder icon into both runtime Swing and `jpackage`.

**Tech Stack:** Java 17, Swing, Maven, PowerShell, GitHub Releases API, GitHub CLI, `jpackage`, WiX, JUnit 5
